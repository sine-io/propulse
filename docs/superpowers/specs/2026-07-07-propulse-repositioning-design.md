# 房脉产品收敛设计：换房决策助手

## 背景

原始项目定位为“房源监测与行情学习网站”，覆盖小区监测、指标解释、学习路线和工具模板。这个方向抓到了用户看房时的信息焦虑，但产品表达偏散，容易变成低配房产数据站或房产知识站。

用户真实场景更具体：有刚需或改善需求，经常在贝壳等平台关注几个小区，但不知道当前价格处于什么状态，不知道自己买得起什么，不知道换房压力多大，也不知道现在是否适合出手。

## 新定位

房脉收敛为面向刚需与改善用户的换房决策助手。

核心问题：

- 我现在能不能买？
- 我能买多少钱？
- 买完压力有多大？
- 目标小区现在贵不贵？
- 现在适合看、等、砍价，还是出手？
- 我如何在使用过程中建立自己的判断方法？

## 核心设计原则

1. 先解决决策焦虑，再讲方法论。
2. 不预测涨跌，只判断信号和压力。
3. 不替用户拍板，但必须给出明确建议。
4. 每个建议都要说明依据。
5. 学习不独立成课，而是嵌入每个判断。
6. 预算、现金流、小区信号必须合并判断。

## 产品主线

### 换房能力

通过用户输入家庭资产、旧房情况、收入、月供、税费、装修和过渡成本，输出安全总价、勉强总价、危险总价、月供压力、资金缺口和换房策略建议。

### 目标小区观察

用户添加目标小区后，围绕挂牌、成交、降价、供应压力和流动性持续观察，输出当前状态和下一步行动建议。

### 出手窗口判断

将用户预算状态与小区行情信号合并，给出“看 / 等 / 砍价 / 出手”的阶段性建议。

### 判断方法

以真实问题组织内容，例如“为什么不能只看挂牌价”“什么是买方窗口”“改善换房为什么要看新旧房价差”，帮助用户在决策过程中建立方法论。

## MVP 范围

MVP 包含：

- 首页
- 换房测算页
- 目标小区页
- 出手窗口页
- 判断方法页
- 工具模板页

暂不做：

- 全国房价预测
- 泛资讯流
- 评论社区
- 房源交易撮合
- 专家直播
- 绝对买卖承诺
