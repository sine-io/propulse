# Issue 编写指南（原子性标准 + 验收片段库）

本指南来自 [2026-07-13 Issue 原子性（INVEST）评审报告](audits/2026-07-13-issue-atomicity-invest-audit.md)。
建卡前请先读完，目标是让每张卡都是「单一关注点 + 单 PR + 可观测验收」的原子看板卡。

## 1. 三段不得雷同

「目标」「当前问题」「验收标准」三段必须内容互异：

- **目标**：要交付什么结果 / 为什么。
- **当前问题**：证据——具体文件、行号、接口，让人能核实问题真实存在。
- **验收标准**：可观测的通过条件——不是把目标再抄一遍。

评审时若验收标准与目标高度重合，直接打回。

## 2. 验收标准写成可观测断言

用 Given/When/Then 或勾选式，每条都能截图或跑测试验证。

反例（复述目标，不可验收）：

> - [ ] 要求显式选择已关注小区，并将 ID 传给 API；刷新后选择稳定。

正例（可观测）：

> - [ ] 请求不含 `neighborhoodId` 时，API 返回 `watchlist-required` 状态，不返回任何默认建议。
> - [ ] 刷新页面后，选择从 URL/本地存储恢复并发起相同 ID 的请求。
> - [ ] 所选小区从观察池移除后，页面进入「重新选择」状态，不静默切换。

## 3. 实现范围禁止样板文字

列出具体文件 / 函数 / 迁移 / OpenAPI 路径，评审据此判断是否越过单 PR 上限。

反例：

> - [ ] 按期望行为完成前端、后端或内容实现。

正例：

> - [ ] `internal/calc/config.go` 新增 `AssumptionConfig`（首付比例/系数/储备/压力线 + 规则版本）。
> - [ ] `internal/calc/service.go` 改为从配置读取，删除内联常量。
> - [ ] 更新 `api/openapi.yaml` calc 响应，补 `ruleVersion` 字段。

## 4. 单一路径，避免工作量漂移

一张卡只能走一条路径：**完整实现** 或 **占位清理/移除**。若两条都要，拆成两张卡。
「既可能补全五篇文章，也可能只保留已完成主题」这类未定路径会让估算在 0.5 天和数天间漂移。

## 5. Epic 与原子任务分轨

- 单个 PR / 1-2 天能完成 → 用 `task.yml`。
- 跨多个 PR / 关注点 → 用 `epic.yml`，拆成子任务，并给 **Epic 级端到端验收**（贯通验收，非子任务堆叠）。
- Epic 不套用原子量表，改按「拆分完备度」自检：覆盖全部交付面、互斥、成链、有端到端验收。

## 6. 可复用验收片段库

在验收标准中按需展开为具体断言。

### 6.1 数据 provenance 五要素

> - [ ] 每条市场数据在页面/接口上都带非空的：来源、采集时间、统计窗口、样本数或覆盖状态、质量提示。
> - [ ] 缺任一要素时按「数据不足」处理，不静默展示为正常值。

### 6.2 状态机六态

> - [ ] loading / empty / error / stale / insufficient / locked 六态各有独立 UI 与测试。
> - [ ] 请求进行中不显示旧数据；失败只显示错误与重试，不回退演示数据。
> - [ ] 无指标时不显示 0 或结论；过期数据带 stale 警告。

### 6.3 鉴权

> - [ ] 无 token / 错 token 访问个人 API 返回结构化 401。
> - [ ] 公共数据匿名可读；个人数据未鉴权不可读。
> - [ ] 会话过期跳转登录，不静默使用旧身份。

### 6.4 计算结果单一来源

> - [ ] 正式结论只来自一次真实计算的返回值，带记录 ID、时间、规则版本。
> - [ ] 接口失败不继续显示为正式或最新报告。
> - [ ] 不存在第二处重复计算或 display override（grep 断言）。

## 7. 依赖关系

- 有上游依赖是正常的（分层构建），如实写 `Blocked by`。
- 相邻 Issue 共享能力时，必须点名归口方并互设 `Blocked by`，防止重复实现或责任真空。
- 链尾聚合型任务在父 Epic 补一条「输入/fixture 改变 → 输出同步变化」的集成验收。
