# PRD v1：房脉 / proppulse

## 1. 背景

刚需和改善用户在买房、换房时，真正困难的不是没有信息，而是无法把信息转成决策。

他们会反复打开贝壳、安居客等平台，看关注小区的房源、价格和降价信息，但常常停留在“看看”的状态：

- 不知道当前价格处于高位还是合理区间
- 不知道挂牌变多、降价变多意味着什么
- 不知道自己的预算到底能买什么
- 不知道月供和现金流压力是否安全
- 不知道现在该等、该看、该砍价，还是该出手

房脉的目标是把买房和换房决策变成一个可解释、可复盘的系统。

## 2. 产品目标

### 业务目标

- 帮用户建立明确的买房 / 换房决策路径
- 帮用户完成预算、压力、小区行情和出手窗口的综合判断
- 让用户在使用过程中逐步建立自己的判断方法
- 形成围绕目标小区的持续观察和复盘习惯

### 用户目标

- 快速知道自己能买多少钱的房
- 明白买完后的月供和现金流压力
- 看懂目标小区当前是偏贵、松动、可砍价还是值得重点看
- 知道下一步该看房、等待、砍价、调整预算还是放弃
- 学会一套不会被中介、博主、亲友轻易带偏的判断方法

## 3. 核心用户

### 主用户

3-12 个月内准备买房或换房的人。

### 典型用户

#### 刚需用户

- 第一次系统看房
- 有首付但不知道买哪里
- 害怕高位接盘
- 需要建立基础判断框架

#### 改善用户

- 已有住房，想卖旧买新
- 关注多个改善小区
- 不知道新旧房价差是否合适
- 担心月供、税费、装修和过渡成本叠加后压力过大

## 4. 核心闭环

```text
输入家庭资产与收入
  ↓
得到安全总价、压力等级、风险边界
  ↓
添加 3-10 个目标小区
  ↓
观察挂牌、成交、降价、供应与流动性信号
  ↓
获得看 / 等 / 砍价 / 出手建议
  ↓
理解判断依据，记录决策过程
  ↓
每周复盘并调整预算、目标小区和行动
```

## 5. MVP 范围

### 5.1 首页

目标：让用户 10 秒内理解网站用途。

核心内容：

- 一句话定位
- 换房测算入口
- 添加目标小区入口
- 示例判断卡片
- 方法论说明入口

### 5.2 换房测算页

目标：让用户知道自己能不能买、能买多少、压力多大。

输入：

- 当前现金
- 旧房估值
- 旧房贷款余额
- 家庭月收入
- 当前月供
- 可接受月供
- 目标总价
- 装修预算
- 税费与中介费预估
- 是否需要过渡租房

输出：

- 安全总价
- 勉强总价
- 危险总价
- 首付缺口
- 月供压力等级
- 现金流风险提示
- 旧房最低安全成交价
- 换房策略建议：先卖后买 / 先买后卖 / 同步推进

### 5.3 目标小区页

目标：让用户看懂一个小区是否值得继续观察。

功能：

- 添加目标小区
- 查看小区基础信息
- 查看价格带
- 查看挂牌量变化
- 查看新增挂牌
- 查看降价房源
- 查看成交价格区间
- 查看流动性和供应压力
- 加入观察池

输出：

- 当前状态：重点看 / 继续观察 / 适合砍价 / 价格偏硬 / 暂不建议追
- 核心原因
- 用户应该关注的价格区间
- 下一步行动建议

### 5.4 出手窗口

目标：把预算和小区信号合并成行动建议。

判断维度：

- 用户预算是否安全
- 目标小区价格是否进入合理区间
- 挂牌与成交是否背离
- 降价频率是否上升
- 供应压力是否增加
- 目标户型是否稀缺
- 替代小区是否更优

输出：

- 看：可以开始实地看房，但不急下定
- 等：价格或预算条件还不合适
- 砍：进入可谈价窗口
- 出手：预算安全且目标房源具备稀缺性或价格优势

### 5.5 判断方法页

目标：让用户在真实决策里学习方法论。

内容围绕问题组织，而不是围绕术语组织：

- 为什么不能只看挂牌价？
- 什么是买方窗口？
- 为什么改善换房要看新旧房价差？
- 为什么月供安全线比总价重要？
- 降价房源变多意味着什么？
- 成交弱但挂牌多，应该怎么看？
- 目标小区和替代小区应该如何对比？

### 5.6 工具模板页

目标：让用户把方法带到站外，形成复盘习惯。

模板：

- 换房预算表
- 目标小区观察表
- 周监测表
- 看房记录表
- 谈价清单
- 决策复盘表

## 6. 指标体系

### 6.1 家庭预算指标

- 可用现金
- 旧房预计净回款
- 贷款余额
- 首付能力
- 税费成本
- 装修成本
- 过渡成本
- 月供收入比
- 安全总价
- 风险总价

### 6.2 小区行情指标

- 挂牌均价
- 成交均价或成交区间
- 挂牌成交差
- 在售套数
- 新增挂牌
- 降价房源数量
- 降价幅度
- 成交周期
- 流动性
- 供应压力

### 6.3 换房决策指标

- 新旧房价差
- 旧房最低安全成交价
- 目标房源合理入手价
- 替代小区性价比
- 买方窗口等级
- 月供压力等级
- 现金流安全边界

## 7. 数据原则

- 优先小区级数据，而不是城市宏观均价
- 优先趋势和变化，而不是孤立价格点
- 优先解释依据，而不是黑箱结论
- 支持用户手动录入和修正关键数据
- 数据不足时必须明确标注，不假装精确
- 不把预测包装成确定性判断

## 8. 风格原则

- 像买方顾问，不像中介
- 像决策工具，不像资讯站
- 像复盘系统，不像玄学预测器
- 语言直接，结论明确，依据透明
- 小白能看懂，进阶用户不嫌浅

## 9. 暂不做

- 全国房价预测
- 专家观点资讯流
- 评论社区
- 房源交易撮合
- 中介 CRM
- 复杂社交系统
- 给用户绝对买卖承诺

## 10. 成功标准

- 新用户 3 分钟内理解产品用途
- 新用户 10 分钟内完成一次换房测算
- 用户能添加至少 3 个目标小区
- 用户能明确说出自己的安全总价和危险总价
- 用户能根据产品建议采取一种行动：看、等、砍价、调整预算或放弃
- 用户一周后愿意回来复盘目标小区变化
